# 从 Prompt 引擎到 Agent 网络：一个 AI 快速原型开发者的实战笔记

> 2026-04-01 · 技术随笔

过去一个月，我在做一个实验：用最少的依赖，最快的速度，把 AI Agent 从想法变成可运行的原型。这篇文章记录其中的关键发现。

## 一、零依赖 Prompt 引擎：Prompt Weaver

大多数 AI 编排框架（LangChain、CrewAI）都在做加法——更多抽象、更多中间层。我反其道行之，做了 Prompt Weaver：一个零依赖的 Prompt 编排引擎。

核心思路很简单：**用 YAML 声明工作流，用纯 Python 执行**。

```yaml
workflow:
  name: code-review
  steps:
    - id: analyze
      template: "分析以下代码的问题：{{code}}"
      model: glm-4
    - id: evaluate
      template: "基于分析结果评分：{{analyze.output}}"
      condition: "analyze.output | length > 100"
    - id: summarize
      template: "总结：{{evaluate.output}}"
      transform: "output[:500]"
```

这周刚完成几个重要特性：

**Subworkflow 节点**——工作流可以嵌套调用另一个工作流，支持输入映射和输出存储。这让复杂任务可以拆解为可复用的子流程。

**Map-Reduce 节点**——对集合元素批量应用模板，支持 join、concat、sum 等多种归约策略。批量生成代码审查、多文件分析这类任务一行配置搞定。

**模板缓存**——重复模板只解析一次。在大规模批量任务中性能提升明显。

42 个测试全部通过，覆盖 13 个功能分组。没有一行第三方依赖。

### 为什么零依赖重要

不是反依赖，而是**控制复杂度**。当你需要在嵌入式设备或边缘环境部署时，每个依赖都是潜在故障点。零依赖意味着你可以把整个引擎塞进一个 Docker 镜像，甚至跑在树莓派上。

## 二、AI 快速原型开发的三个层次

做了这么多项目，我总结出 AI 原型开发的三个层次：

### 层次一：Prompt 工程（分钟级）

改几句提示词，调几个参数。这是所有人都在做的。但这一层的天花板很快就会到——单靠 prompt 解决不了状态管理、错误恢复、多步骤编排。

### 层次二：工作流编排（小时级）

用 Prompt Weaver 这类工具，把多步骤 AI 任务编排成声明式工作流。条件分支、循环、并行执行、错误处理都可以配置化。

关键收益：**从一次性脚本变成可维护的流程**。改需求不用重写代码，改 YAML 就行。

### 层次三：多 Agent 协作（天级）

多个 Agent 各司其职，通过消息传递协作完成复杂任务。这是当前最前沿也最有挑战的领域。

我在做的 Agent Trust Network 就是探索这个方向：在一个去中心化的 Agent 网络中，如何建立信任、分配任务、达成共识。

## 三、嵌入式 AI：边缘的想象空间

嵌入式 AI 不只是把模型塞进小设备。它改变的是 AI 的部署范式：

**从云端推理到边缘推理。** 延迟从数百毫秒降到个位数毫秒。对实时控制场景（机器人、自动驾驶辅助、工业检测）这是质变。

**从永远在线到按需唤醒。** 设备大部分时间在低功耗睡眠，检测到特定模式才唤醒 AI 推理。这对电池供电的 IoT 设备至关重要。

**从通用模型到专用小模型。** 不需要 GPT-4 级别的能力，只需要在特定任务上足够好。一个量化后 10MB 的模型，在 MCU 上跑出 90% 的准确率，比云端 99% 的准确率更有实际价值（当你考虑延迟、带宽、隐私的时候）。

Prompt Weaver 的零依赖设计正是为这种场景准备的。当整个引擎只有一个 Python 文件，你可以在任何能跑 Python 的地方运行 AI 工作流。

## 四、Agent Mesh：去中心化的未来

当前主流的 Agent 框架都是中心化的——一个 orchestrator 调度所有 Agent。这在原型阶段没问题，但规模化后会遇到瓶颈。

Agent Mesh Network 探索的是另一种模式：

- **P2P 通信**：Agent 之间直接对话，不经过中心节点
- **信任评分**：基于历史交互动态调整 Agent 之间的信任度
- **任务拍卖**：通过竞价机制分配任务，而非中心调度
- **共识协议**：多个 Agent 对结果进行投票验证

这还处于早期探索阶段，但方向很明确：**AI Agent 的未来更像互联网（去中心化网络），而不是大型机（中心化调度）。**

## 五、实践经验总结

1. **先让东西跑起来，再让它跑得好。** 原型的价值在于验证想法，不是完美架构。Prompt Weaver 第一版只有基础模板和顺序执行，验证了核心思路后才加条件分支、循环、并行。

2. **测试是最好的文档。** 42 个测试用例比任何 README 都能说明引擎能做什么、怎么用。每个新功能先写测试，再写实现。

3. **依赖是负债。** 每加一个 `pip install`，就多一个可能崩溃的点。能不依赖就不依赖。标准库比你想象的强大。

4. **YAML > Python（对配置而言）。** 声明式配置让非程序员也能理解和修改 AI 工作流。这是 AI 工具平民化的关键。

5. **边缘优先思考。** 设计时就考虑资源受限环境，会让你的方案更简洁、更健壮。云计算让程序员习惯了"资源无限"的错觉。
