### 2.1 AI 快速原型开发的方法论

**核心原则**:
1. **快速验证 > 完美实现**
   - MVP 思维
   - 6-7 小时从想法到部署
   
2. **AI 是加速器，不是替代**
   - 让 AI 承担重复性工作
   - 人专注于创造性和决策
   
3. **用户反馈是金**
   - 尽早获得真实反馈
   - Build-Measure-Learn 循环
   
4. **简单优于复杂**
   - YAGNI 原则
   - 重构而非预设计
   
5. **测试不可跳过**
   - TDD 保证质量
   - Ralph Mode 自动验证

**完整工作流**:
```
Phase 1: Ideation (15-30 min)
  → 定义问题、用户、价值主张
  
Phase 2: Brainstorming (30-60 min)
  → 使用 brainstorming skill
  → 技术选型、架构决策
  
Phase 3: Planning (15-30 min)
  → 创建 SPEC.md
  → 分解任务、定义验收标准
  
Phase 4: Building (1-4 hours)
  → Ralph Mode / Subagents
  → TDD 开发、自动化测试
  
Phase 5: Validation (30-60 min)
  → 功能测试、用户测试
  → 收集反馈
  
Phase 6: Iteration (持续)
  → 分析反馈、优先级排序
  → 快速改进
```

**实战经验**:
- **CodeReview AI**: 6 小时从想法到部署

### 2.2 边缘 AI Agent 的协议设计

**TAP 协议的创新点**:

1. **极简设计**
   ```typescript
   interface TAPMessage {
     v: 1;              // 版本
     id: string;        // 8 chars
     t: number;         // 时间戳
     a: TAPAction;      // 动作
     p?: any;           // 负载
   }
   ```
   - 核心 < 10KB 代码
   - 无状态设计
   
2. **渐进式智能**
   ```
   Level 0: Pure Reflex (无条件反射)
     → 本地决策，无需云端
   
   Level 1: Cloud-Assisted (云端辅助)
     → 感知 → 分析 → 指令
   
   Level 2: Predictive (预测性)
     → 云端学习 → 本地预测
   
   Level 3: Collaborative (协作性)
     → 多设备协调
   ```
   
3. **电量感知调度**
   - Critical: 1 小时心跳，仅 ping
   - Low: 10 分钟心跳，ping + sense
   - Normal: 1 分钟心跳，全部功能
   - High: 30 秒心跳，全部功能
   
4. **离线降级**
   ```typescript
   const FALLBACK_RULES = {
     'temperature': {
       condition: (v) => v > 30,
       action: 'fan_on',
       priority: 'high'
     }
   };
   ```

**应用场景**:
- 智能家居（温控、照明、安防）
- 农业物联网（土壤监测、自动灌溉）
- 工业监控（设备状态、预测性维护）
- 环境监测（空气质量、水质）

### 2.3 记忆系统的演进与对比

**三种方案对比**:

| 特性 | MEMORY.md | Memori | KuzuDB | ontology |
|------|-----------|--------|--------|----------|
| 类型 | 文件系统 | SQL Native | Graph DB | Typed KG |
| 复杂度 | 极简 | 中等 | 高 | 高 |
| 查询能力 | 手动 | SQL | Cypher | 类型化查询 |
| 性能 | - | 81.95% 准确率 | 374x 加速 | 语义丰富 |
| 成本 | 免费 | 67% 降低 | - | - |
| OpenClaw 集成 | 手动 | 官方插件 | - | - |

**选择建议**:
- **个人/小项目**: MEMORY.md（简单、免费、完全控制）
- **生产系统**: Memori（自动、结构化、成本优化）
- **复杂关系**: KuzuDB（图查询、multi-hop）
- **语义丰富**: ontology（类型化、知识图谱）

**演进路径**:
```
MEMORY.md (手动、文件系统)
  → Memori (自动、SQL Native)
    → KuzuDB (图关系、复杂查询)
      → ontology (类型化知识图谱)
```
