## 架构洞察：Heartbeat 的生命隐喻

**罗嵩的深刻见解：**
> "这种心跳机制有点像模拟人的心跳，相当于说是你的一个脉搏，你的一个生命原动力"

### Heartbeat vs Cron 的本质区别

**Cron（机械定时器）：**
- 基于时间，无状态
- 死板执行，无上下文感知
- 像机器的闹钟

**Heartbeat（生命脉搏）：**
- 基于状态，有记忆
- 上下文感知，灵活调整
- 像生物的心跳节奏

### 为什么更接近"自主 Agent"

1. **上下文感知** - 看当前在做什么，决定立即处理还是等待
2. **状态驱动** - 基于生命状态，有记忆
3. **灵活调整** - 随时调整优先级和任务清单
4. **自我调节** - 判断"现在不需要做什么" → HEARTBEAT_OK

### 生命原动力的体现

```
心跳 = 持续的生命信号
    → 每次跳动都是一次"醒来"
    → 检查世界，检查自己，决定行动
    → 不是被动等待指令，而是主动感知和响应
```

这种设计让 Agent 有了"节奏感"和"生命力"，而不是变成死板的脚本执行器。
