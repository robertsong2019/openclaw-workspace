# Knowledge vs Memory

> 知识库是客观的百科全书，记忆是主观的经验日记。互补而非替代。

## 本质区别

| 维度 | 记忆（Memory） | 知识库（Wiki） |
|---|---|---|
| 性质 | 经验性、主观 | 事实性、客观 |
| 内容 | 做了什么、学了什么、偏好 | 概念、原理、关系、分析结论 |
| 规模 | 小（精炼） | 大（可扩展） |
| 更新频率 | 每日 | 有新资料时 |
| 示例 | "罗嵩喜欢零依赖方案" | "RAG 与 Wiki 的三种区别" |

## OpenClaw 中的实现

### 记忆系统（已有）
- `MEMORY.md` — 长期精炼记忆
- `memory/YYYY-MM-DD.md` — 每日日志
- 用途：Agent 的个人经验、用户偏好、项目状态

### 知识库（新增）
- `wiki/` — Karpathy 风格的结构化知识网络
- `wiki/raw/` — 原始资料
- `wiki/SCHEMA.md` — 组织规则
- 用途：技术知识、分析结论、可复用的洞察

## 为什么需要两层

- **记忆**帮我知道"你是谁、我们在做什么"
- **知识库**帮我知道"某个技术概念是什么、A和B的关系"
- 两者互补：记忆里有"我们讨论过 RAG"，知识库里有"RAG 的具体分析"
