# Agent Communication Entropy

> 组织沟通成本是熵增的，线性加 Agent 导致沟通成本平方增长

## 核心问题

N 个 Agent → N(N-1)/2 条沟通线。加人不一定加速。

## 降低沟通熵的五个原则

### 1. 约定大于沟通（Convention over Communication）
- 输入输出格式、命名规则、接口契约预先固化
- Agent 间 80% 的"沟通"其实是在对齐格式，应该预先约定

### 2. 共享状态替代点对点消息
- 所有人读写同一个黑板，而非 A→B→C 逐个传话
- 沟通成本从 O(N²) 降到 O(N)

### 3. 最小权限 + 领域隔离
- 每个 Agent 只知道输入从哪来、输出给谁、边界是什么
- 像微服务：独立部署、接口通信、内部自治

### 4. 事件驱动而非轮询
- 完成后发布事件，需要的 Agent 自然响应
- 不要每隔 N 秒问"你好了吗？"

### 5. 分层治理
- 80% 工作在执行层自治完成，只有 20% 需要跨层沟通
- 执行层（自治）→ 协调层（边界交互）→ 战略层（单点决策）

## 金句

> 最好的沟通是不沟通。能用约定解决的不要用协议，能用共享状态解决的不要用消息，能自治完成的不要协调。

## 现实参照

Linux 内核几万贡献者的协调方式：
- 代码规范（约定）
- Git 仓库（共享状态）
- Maintainer 分层（分层治理）
- Patch 提交（事件驱动）
