# Knowledge Base Schema

> 告诉 LLM 如何组织、维护这个 Wiki

## 组织规则

### 文件命名
- 全小写，连字符分隔：`agent-engineering.md`
- 每个文件对应一个主题/概念/实体

### 页面结构
每个 Wiki 页面遵循：
```markdown
# 页面标题
> 一句话摘要

## 核心内容
...

## 关联
- 相关 Wiki 页面链接
- 来源：原始资料引用

---
_最后更新：YYYY-MM-DD_
```

### 交叉引用
- 页面之间用 `[[wiki-link]]` 格式双向链接
- 新建页面必须更新 `index.md`
- 新知识必须检查并更新所有相关页面

## 三种操作

### Ingest（录入）
1. 读取新资料
2. 提取关键信息
3. 创建新页面或更新已有页面
4. 更新 index.md
5. 更新所有关联页面（每个来源可能牵动 10-15 个页面）

### Query（查询）
1. 先读 index.md 定位相关页面
2. 精读相关页面内容
3. 综合回答
4. 好的回答回存为新的 Wiki 页面（知识复利）

### Lint（体检）
- 找页面间矛盾
- 标记过时信息
- 找孤儿页面（没有入链）
- 补充缺失的交叉引用
- 建议新的研究方向

## 边界
- **知识库**（wiki/）：事实性、客观的 → 你正在读的
- **记忆**（MEMORY.md / memory/）：经验性、主观的 → Catalyst 的记忆
- **原始资料**（wiki/raw/）：不可变的原始数据
